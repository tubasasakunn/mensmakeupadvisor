# アプリのプライバシー ── ASC での回答シート（手動・初回のみ）

「アプリのプライバシー」セクションは App Store Connect API / fastlane では
設定できない（fastlane の `upload_app_privacy_details_to_app_store` は
Apple ID パスワードログイン前提で CI に組み込めない）。
**Admin が ASC の画面から 1 回だけ**回答する。所要 2 分。

## 手順

1. https://appstoreconnect.apple.com/apps → 対象アプリ →
   左メニュー「アプリのプライバシー」
2. データ収集の有無を回答する。
   - データを一切オフデバイスへ送らないアプリ：
     「**いいえ、この App からデータを収集しません**」→ 右上「公開」。
   - 何か送る場合：収集するデータタイプ（識別子・使用状況・連絡先など）を
     正直に申告する。下の「根拠」を自分のアプリの実装に合わせて書き換える。

## 根拠（このアプリの場合・要編集）

- すべてのデータは端末内に保存。外部サーバーへの送信なし。
- アナリティクス・広告 SDK なし。
- iCloud 同期（任意 ON）は Apple のユーザー私有データベース（CloudKit
  プライベート DB）への保存であり、開発者がアクセスできる「データ収集」に
  該当しない。
- 位置情報を使う場合は、端末内のみで使い送信しないなら収集に当たらない。

→ Apple の定義する「収集（オフデバイスへの送信）」に該当するデータの有無を、
  自分のアプリの実装で確認して回答する。

## 年齢制限の補足

年齢制限指定は CI が `rating.json` から**全設問を自動反映**する
（2025 年改定の新設問──広告・メッセージ/チャット・UGC・ペアレンタル
コントロール・健康トピック・年齢確認──は API 上すべて boolean で、
ASC は PATCH 時に全件必須として要求する）。
反映後に ASC 画面で意図どおりのレーティング（例「4+」）になっていることだけ確認する。
