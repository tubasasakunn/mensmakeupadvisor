# Architecture Decision Records (ADR)

このプロジェクトの「なぜそうなっているか」を記録する場所。コードやコメントから
読み取りにくい**意思決定の背景と却下した代替案**をここに残す。

## 運用ルール

- 1 決定 = 1 ファイル。`NNNN-短いケバブケース.md` で連番を振る。
- 一度 Accepted になった ADR は**書き換えない**。決定を覆すときは新しい ADR を起こし、
  古い方の Status を `Superseded by NNNN` に更新する（本文は残す）。
- 「再考の条件」を必ず書く。決定は当時の制約の関数なので、制約が消えたら見直せるようにする。
- 書くべきタイミング：アーキテクチャ・データモデル・依存ライブラリ・並行性方針など、
  後から「なんでこうなってるの？」と聞かれそうな決定をしたとき。
  `/adr` スキル（`.claude/skills/adr/`）でテンプレートから起こせる。
