## 0. トーン&マナー（全投稿共通）
- 芯：**「盛る」ではなく「整える」／気づかれない範囲がデフォルト**。別人化はしない＝「調子のいい日の自分」が目標。
- ブランド：ダーク×ウォーム高級感、wordmark "Tone"、accent warm red。アプリは**端末内完結（画像を外に送らない）**。
- メイクは理屈＝**光と影**（明るく=前に出す/暗く=引っ込める）。覚えれば誰でも再現できる、が一貫メッセージ。

## 1. 統計フック（1枚目に強い・出典付き）
| 数字 | 意味 | 出典 |
|---|---|---|
| **55%** | 第一印象は見た目で決まる比率 | Mehrabian, UCLA (1971) |
| **0.1秒** | 相手が値踏みするまでの時間 | Willis & Todorov, Princeton (2006) |
| **71.9%** | 20代男性で美容に興味がある割合（＝あなたは変人じゃない） | ホットペッパービューティーアカデミー(2023) |
| **3,000回** | 1日に自分の顔を見る回数 | 研究推計 |
| **13%** | 外見上位20%が下位20%より高い年収（学歴差より大） | Hamermesh "Beauty Pays"(2011) |
| **38% / 42%** | 交際相手の外見を重視する男性/女性の割合 | マッチングアプリ調査(2023) |
| **21日** | 習慣が「当たり前」になるまでの日数 | Maxwell Maltz(1960) |
- 概念フック：**ハロー効果**(見た目が整う→能力まで良く見える)／**自信の連鎖**(顔→姿勢→声→評価→自信)／**外見プレミアム**。

## 2. 理論（カルーセルの中身に最適）
- **整って見える比率**＝黄金比/三分割/五分割。顔タイプごとに正解が違う。
- 目指すのは**あなたの最良版**（別人を真似ると失敗＝コスプレ）。
- 道具は**光と影だけ**：明るく=大きく近く見える／暗く=小さく遠く見える。色も光と影の仲間(赤橙=光/青灰=影、青ヒゲは青を打ち消す)。
- **顔タイプ（AIが判定）**：卵型(バランス良)／面長(横に広く見せる)／丸顔(縦に長く＋シェーディング)／ベース型(エラ→フェイスラインのシェーディング最重要)。
- 光と影の具体例：目が近い→目頭内側に光／遠い→目頭に影。小顔→頬骨下〜顎に影。鼻→鼻筋に光＋両側に影。おでこ広い→生え際に影。

## 3. 5ステップ（howto・番号リスト/info に最適）
1. **Base（ベース）**：青ヒゲ・赤み・ニキビ跡=ノイズを消す。BBクリームは肌色より一段明るい(オークル20)、1円玉大を5点置き→外へ伸ばす。
2. **Highlight（ハイライト）**：おでこ中央・鼻筋・頬骨高い所・あご先に少量。骨を立て重心を上げる＝若々しさ。小指の先量、ポンポン。
3. **Shadow（シェーディング）**：こめかみ・頬下・フェイスライン。肌色より2〜3段暗いブラウン(グレー避ける)。輪郭が引き締まる。
4. **Eyes（目元）**：会話で一番見られる＝ROI最大。まつ毛の隙間を埋める(濃いブラウン)＋涙袋に細くハイライト。
5. **Brows（眉）**：実は一番効く＆社会的に完全OK＝ノーリスク。毛量を揃える→ペンで眉頭薄く眉尻濃く→眉マスカラ。

## 4. 続け方・よくある失敗（共感/ノウハウ）
- **週3回(月水金)・10分以内・道具3つ**から。道具は最小限：Day1 BBクリーム→Day8 眉ペン→Day30 シェーディング。
- 失敗①全部やる→やり過ぎ感(最初は1〜2ステップ)。②量が多すぎ(自然光で確認、1円玉大)。③バレたくない強すぎて効果ゼロ(鏡から60cmで判断、他人は気にしてない)。
- 成功のサイン：「今日なんかいい感じ」「肌きれい」と言われる（「化粧してる」とは言われない）。

## 5. 投稿の切り口バックログ（1投稿=1テーマ・1要素実験）
- 統計フック型：「第一印象は55%が見た目」「0.1秒で判定」「20代男性の72%が美容に興味」
- 顔タイプ型：「あなたは卵型？面長？」「丸顔がやるべき2つ」「ベース型はここだけ」
- 理論型：「メイク＝光と影、これだけ」「色も光と影の仲間」
- 5ステップ型：「やることは5つだけ」「眉が一番効く（ノーリスク）」「ROI最大は目元」
- 続け方/失敗型：「週3回・10分でいい」「初心者がやりがちな3失敗」「Day1はBBクリームだけ」
- ビフォーアフター/エモ型：「半年後の月曜の朝」「盛らずに整えるだけ」
→ 既存と被らないよう LEARNINGS の実験ログで消化済みテーマを管理し、毎回1テーマだけ。

# SNS 投稿（カルーセル画像）の作成
> **デザイン品質の正本は `/carousel-craft` スキル**（`~/.claude/skills/carousel-craft/`）。表紙含め**全スライドに用意済みの実素材**（実app画面/footage/素材バンク）を敷き、フック・タイポ・配色・セーフゾーンを `DESIGN_SPEC.md` の規範で設計、生成後は `scripts/qa.py`＋**目視**でレビュー。素材在庫とspecレシピは `MATERIALS.md`。**灰色グラデ/ベタ塗りだけの背景を作らない**。


TikTok（1080x1920 / 9:16）と Lemon8（1080x1440 / 3:4）の**複数画像投稿**を 1 本作る。
仕組みの正本は `post/README.md`、描画エンジンは `post/_brand.py`、コンテンツ定義は
`post/build_posts.py`（画像）と `post/build_index.py`（文言）。ブランド色・ワードマークは
`appstore.config.json`。ここは「次の 1 本を足す手順」だけ。

## 0. 準備

依存は `Pillow`, `numpy`。見出しフォント（Noto Sans/Serif JP）は `build_posts.py` が
実行時に `/tmp` へ取得する（要ネットワーク）。

## 1. 投稿のコンセプトを決める

- アプリのコンセプト・判断軸に整合する切り口か確認する。
- 既存投稿と**角度を変える**（共感ノウハウ / 映え・プロダクト / プライバシー・エモ など）。
- 構成テンプレ：**表紙（フック）→ 共感/日常 → アプリ実画面（証拠）→ ノウハウ/機能 → CTA**。
- プラットフォーム仕様（サイズ・枚数・タグ数・安全域）は `post/README.md` の表が正本。
  古くなっていそうなら WebSearch で確認して README も更新する。

## 2. （任意）フッテージを用意する

表紙・共感スライドの背景は `material/footage/<name>.(jpg|png)` を `bg` で参照する。
投稿ごとに表紙の背景を変えると 1 枚目が差別化される。生成 AI で作るなら
`material/footage/PROMPTS.md` の方針（縦9:16・顔なし・暖色・文字なし）に従う。
無ければアクセント単色のプレースホルダで埋まる。

**新規生成のほか、ルートの汎用素材バンク `/home/tubasasakun/android/material/` から流用してもよい**
（リポ内 `material/footage/` とは別物）。`index.json`（`{name(uuid), prompt, tags[]}`）を
タグ検索し（例: `tone`/`メンズ`/`縦長`）、画像 `images/<name>.jpg` を `material/footage/<name>.jpg`
にコピーして `bg` から参照する。ダーク×ウォーターの世界観に合うもの・縦長を選ぶ。
```bash
python3 - <<'PY'
import json; b="/home/tubasasakun/android/material"
want={"tone","縦長"}  # 欲しいタグ（例: メンズ系×縦長）
for e in [x for x in json.load(open(b+"/index.json")) if want <= set(x["tags"])][:20]:
    print(b+"/images/"+e["name"]+".jpg", e["tags"])
PY
```

## 3. スライドを定義する（画像）

`post/build_posts.py` の `POSTS` に `"postN"` を追加する。スライド 5 種
（`cover` / `photo` / `shot` / `info` / `cta`）のフィールドは `post/README.md` の表を参照。

- `accent` は投稿単位で 1 色（`appstore.config.json` の `brand.accents` のキー）。個別上書き可。
- `shot` の緑領域はフッテージで差し替えられる（`footage` キー）。
- TikTok は**下 1/3 に文字を置かない**（renderer が安全域を吸収するが行数を増やしすぎない）。

生成して**必ず目視確認**する（Read で画像を開く）：

```bash
cd post && python3 build_posts.py
```

崩れ・はみ出し・読みにくさがあれば文言量や `accent` を調整して再生成。
代表として cover / shot / info / cta と、3:4 が最もタイトな `lemon8` を確認する。

## 4. 本文とハッシュタグを書く（文言）

`post/build_index.py` の `COPY` に `"postN"` を追加（`tiktok` と `lemon8` 両方）：

- **TikTok**：冒頭フック→本文→CTA。タグ 3〜5（関連性重視）。
- **Lemon8**：掴み / 箇条書き▶ / まとめ / CTA。タグ最大 10、大・中・小を **3:4:3** で混ぜる。
- 直リンクは貼らず「プロフィールのリンクから」へ誘導。

```bash
cd post && python3 build_index.py
```

画像リストは実ファイルから自動収集し、`manifest.json` も更新される。
